# 🎯 Spring AOP — Aspect-Oriented Programming

AOP lets you define cross-cutting concerns (logging, security, transactions, caching) separately from business logic. Spring AOP uses **runtime proxies** — no special compiler needed.

---

## 1. Core Concepts

| Term | Definition | Example |
|------|-----------|---------|
| **Aspect** | A module encapsulating a cross-cutting concern | Logging aspect, security aspect |
| **Join Point** | A point during execution where an aspect can be applied | Method execution (Spring AOP only supports method-level) |
| **Advice** | Action taken at a join point | Log before method, measure timing around method |
| **Pointcut** | Expression that selects join points | "All methods in `@Service` classes" |
| **Target Object** | The object being advised (proxied) | Your `OrderService` instance |
| **Proxy** | The AOP-created wrapper around the target | JDK dynamic proxy or CGLIB proxy |
| **Weaving** | Linking aspects with target objects | Runtime weaving (Spring default) |

### How Spring AOP Works Internally

```
Client code calls orderService.placeOrder()
         │
         ▼
┌─────────────────────┐
│    AOP Proxy         │  ← Spring creates this at startup
│  (CGLIB subclass     │
│   or JDK dynamic)    │
│                      │
│  1. Before advice    │
│  2. target.placeOrder()  ← actual method
│  3. After advice     │
│  4. Return / throw   │
└─────────────────────┘
```

### JDK Dynamic Proxy vs CGLIB Proxy

| Aspect | JDK Dynamic Proxy | CGLIB Proxy |
|--------|-------------------|-------------|
| Works with | Interfaces only | Classes (creates subclass) |
| Default when | Bean implements interface (Spring 5-) | Always (Spring Boot default) |
| Final classes | ✅ Works (proxies interface) | ❌ Cannot subclass final class |
| Final methods | ✅ Works | ❌ Cannot override final methods |
| Performance | Slightly slower (reflection) | Slightly faster (generated bytecode) |

:::tip Interview Insight
Spring Boot defaults to **CGLIB proxies** (even when interfaces exist) since Spring Boot 2.0. This is configured via `spring.aop.proxy-target-class=true` (default). CGLIB creates a subclass of your bean — this is why `final` classes/methods cannot be proxied by Spring AOP.
:::

## 2. Advice Types

### `@Before` — Runs Before the Method

```java
@Aspect
@Component
public class LoggingAspect {

    @Before("execution(* com.example.service.*.*(..))")
    public void logBefore(JoinPoint joinPoint) {
        log.info("Calling: {}.{}() with args: {}",
            joinPoint.getTarget().getClass().getSimpleName(),
            joinPoint.getSignature().getName(),
            Arrays.toString(joinPoint.getArgs()));
    }
}
```

### `@AfterReturning` — Runs After Successful Return

```java
@AfterReturning(
    pointcut = "execution(* com.example.service.OrderService.placeOrder(..))",
    returning = "order"
)
public void logOrderPlaced(JoinPoint joinPoint, Order order) {
    log.info("Order placed: id={}, total={}", order.getId(), order.getTotal());
}
```

### `@AfterThrowing` — Runs After Exception

```java
@AfterThrowing(
    pointcut = "execution(* com.example.service.*.*(..))",
    throwing = "ex"
)
public void logException(JoinPoint joinPoint, Exception ex) {
    log.error("Exception in {}.{}(): {}",
        joinPoint.getTarget().getClass().getSimpleName(),
        joinPoint.getSignature().getName(),
        ex.getMessage());
}
```

### `@After` — Runs After Method (Finally Block)

```java
@After("execution(* com.example.service.*.*(..))")
public void logAfter(JoinPoint joinPoint) {
    // Runs regardless of success or exception (like finally)
    log.debug("Completed: {}", joinPoint.getSignature().getName());
}
```

### `@Around` — Full Control (Most Powerful)

```java
@Around("execution(* com.example.service.*.*(..))")
public Object measureTime(ProceedingJoinPoint pjp) throws Throwable {
    long start = System.nanoTime();
    try {
        Object result = pjp.proceed(); // call the actual method
        return result;
    } finally {
        long duration = (System.nanoTime() - start) / 1_000_000;
        log.info("{}.{}() took {}ms",
            pjp.getTarget().getClass().getSimpleName(),
            pjp.getSignature().getName(), duration);
    }
}
```

### Advice Execution Order

```
@Around (before proceed)
  └── @Before
        └── Target Method
  └── @AfterReturning (if success) OR @AfterThrowing (if exception)
  └── @After (always)
@Around (after proceed)
```

## 3. Pointcut Expressions

### `execution()` — Method Execution

```
execution(modifiers? return-type declaring-type? method-name(param-types) throws?)
```

| Expression | Matches |
|-----------|---------|
| `execution(* com.example.service.*.*(..))` | Any method in service package |
| `execution(public * *(..))` | Any public method |
| `execution(* save*(..))` | Any method starting with "save" |
| `execution(String com.example.*.find*(..))` | Methods returning String, starting with "find" |
| `execution(* *(.., String))` | Methods with String as last parameter |
| `execution(* com.example..*.*(..))` | Any method in com.example and sub-packages |

### Annotation-Based Pointcuts

```java
// Match methods annotated with @Transactional
@Before("@annotation(org.springframework.transaction.annotation.Transactional)")

// Match classes annotated with @Service
@Before("@within(org.springframework.stereotype.Service)")

// Match classes annotated with @RestController
@Before("@target(org.springframework.web.bind.annotation.RestController)")
```

### Custom Annotation Pointcut (Most Maintainable)

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Audited {
    String action();
}

@Aspect
@Component
public class AuditAspect {
    @Around("@annotation(audited)")
    public Object audit(ProceedingJoinPoint pjp, Audited audited) throws Throwable {
        log.info("Audit: action={}, user={}", audited.action(), getCurrentUser());
        Object result = pjp.proceed();
        log.info("Audit: completed action={}", audited.action());
        return result;
    }
}

// Usage
@Service
public class AccountService {
    @Audited(action = "TRANSFER_FUNDS")
    public void transfer(Long fromId, Long toId, BigDecimal amount) { ... }
}
```

### Combining Pointcuts

```java
@Aspect
@Component
public class SecurityAspect {

    @Pointcut("execution(* com.example.service.*.*(..))")
    private void serviceLayer() {}

    @Pointcut("execution(* com.example.repository.*.*(..))")
    private void repositoryLayer() {}

    @Pointcut("@annotation(com.example.annotation.Secured)")
    private void securedMethods() {}

    // Combine with &&, ||, !
    @Before("serviceLayer() && securedMethods()")
    public void checkSecurity(JoinPoint jp) { ... }

    @Before("serviceLayer() || repositoryLayer()")
    public void logAllDataAccess(JoinPoint jp) { ... }
}
```

## 4. Real-World AOP Examples

### Retry Mechanism

```java
@Target(ElementType.METHOD)
@Retention(RetentionPolicy.RUNTIME)
public @interface Retryable {
    int maxAttempts() default 3;
    long delayMs() default 1000;
    Class<? extends Exception>[] retryOn() default {Exception.class};
}

@Aspect
@Component
public class RetryAspect {

    @Around("@annotation(retryable)")
    public Object retry(ProceedingJoinPoint pjp, Retryable retryable) throws Throwable {
        int attempts = 0;
        Exception lastException = null;

        while (attempts < retryable.maxAttempts()) {
            try {
                return pjp.proceed();
            } catch (Exception e) {
                lastException = e;
                boolean shouldRetry = Arrays.stream(retryable.retryOn())
                    .anyMatch(clazz -> clazz.isInstance(e));
                if (!shouldRetry) throw e;

                attempts++;
                if (attempts < retryable.maxAttempts()) {
                    Thread.sleep(retryable.delayMs() * attempts);
                    log.warn("Retry {}/{} for {}", attempts, retryable.maxAttempts(),
                        pjp.getSignature().getName());
                }
            }
        }
        throw lastException;
    }
}
```

### Rate Limiting

```java
@Aspect
@Component
public class RateLimitAspect {

    private final Map<String, RateLimiter> limiters = new ConcurrentHashMap<>();

    @Around("@annotation(rateLimit)")
    public Object rateLimit(ProceedingJoinPoint pjp, RateLimit rateLimit) throws Throwable {
        String key = pjp.getSignature().toShortString();
        RateLimiter limiter = limiters.computeIfAbsent(key,
            k -> RateLimiter.create(rateLimit.permitsPerSecond()));

        if (!limiter.tryAcquire(rateLimit.timeoutMs(), TimeUnit.MILLISECONDS)) {
            throw new RateLimitExceededException("Rate limit exceeded for " + key);
        }
        return pjp.proceed();
    }
}
```

## 5. Spring AOP vs AspectJ

| Feature | Spring AOP | AspectJ |
|---------|-----------|---------|
| Weaving | Runtime (proxy-based) | Compile-time or load-time |
| Join points | Method execution only | Method, constructor, field access, static initializer |
| Pointcuts | Subset of AspectJ syntax | Full AspectJ syntax |
| Self-invocation | ❌ Not intercepted | ✅ Intercepted |
| Performance | Slight overhead per call | Near-zero after weaving |
| Setup | Zero config (Spring Boot) | Requires AspectJ compiler or agent |
| Final classes | ❌ Cannot proxy | ✅ Can weave |

:::tip Interview Insight
If asked "When would you use AspectJ over Spring AOP?":
1. You need to intercept **field access** or **constructor calls**
2. You need to intercept **self-invocations** (method A calling method B in the same class)
3. You need AOP on **final classes**
4. Performance is critical — AspectJ has zero runtime overhead
:::

## Summary — Key Takeaways for Interviews

| Topic | What to Know |
|-------|-------------|
| AOP core concepts | Aspect, Advice, Pointcut, Join Point, Proxy |
| Proxy types | CGLIB (default in Boot — subclass) vs JDK (interface-based) |
| Advice types | `@Before`, `@After`, `@AfterReturning`, `@AfterThrowing`, `@Around` |
| `@Around` | Most powerful — controls whether method executes via `proceed()` |
| Pointcut expressions | `execution(* com.example..*.*(..))` — method pattern matching |
| Custom annotations | `@annotation(myAnnotation)` — cleanest pointcut strategy |
| Self-invocation | Same-class calls bypass proxy → AOP advice not applied |
| Spring AOP vs AspectJ | Spring = runtime proxies (method only); AspectJ = compile-time (everything) |
| Common uses | Logging, security, transactions, caching, retry, rate limiting |
